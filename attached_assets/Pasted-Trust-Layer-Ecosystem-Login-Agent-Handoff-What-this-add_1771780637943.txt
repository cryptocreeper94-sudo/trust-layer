Trust Layer Ecosystem Login — Agent Handoff
What this adds: A "Sign in with Trust Layer" button on the login page that lets members from any DarkWave ecosystem app (Happy Eats, TrustHome, Signal, TrustVault, Verdara, etc.) sign in using their Trust Layer ID or ecosystem email + either their full password or a short PIN.

STEP 1: Schema — Add columns to your users table
In your shared schema file (e.g. shared/schema.ts), add two columns to the users table:

ecosystemPinHash: text("ecosystem_pin_hash"),
ecosystemApp: text("ecosystem_app"),
ecosystemPinHash stores a bcrypt hash of a 4-8 digit PIN for quick ecosystem login.
ecosystemApp stores which ecosystem app the user originally came from (e.g. "Happy Eats", "TrustHome").
You should already have trustLayerId (format: tl-xxxx-xxxx) on your users table. If not, add:

trustLayerId: text("trust_layer_id"),
Then push/migrate your database schema.

STEP 2: Storage — Add a lookup method
In your storage layer, add a method to find users by Trust Layer ID:

async getUserByTrustLayerId(trustLayerId: string): Promise<User | undefined> {
  const [user] = await db.select().from(users).where(eq(users.trustLayerId, trustLayerId));
  return user;
}
STEP 3: Backend — Add the /api/auth/ecosystem-login endpoint
Add this route in your auth setup. It accepts { identifier, credential } where identifier is a Trust Layer ID (tl-xxxx-xxxx) or email, and credential is either a full password or a numeric PIN:

app.post("/api/auth/ecosystem-login", async (req: Request, res: Response) => {
  try {
    const { identifier, credential } = req.body;
    // Input validation — prevents crashes from non-string payloads
    if (!identifier || !credential || typeof identifier !== "string" || typeof credential !== "string") {
      return res.status(400).json({ message: "Trust Layer ID or email and credential are required" });
    }
    const trimmedId = identifier.trim();
    const trimmedCred = credential.trim();
    if (!trimmedId || !trimmedCred) {
      return res.status(400).json({ message: "Trust Layer ID or email and credential are required" });
    }
    // Lookup: try Trust Layer ID first, then fall back to email
    let user;
    if (trimmedId.startsWith("tl-")) {
      user = await storage.getUserByTrustLayerId(trimmedId);
    }
    if (!user) {
      user = await storage.getUserByEmail(trimmedId.toLowerCase());
    }
    if (!user) {
      return res.status(401).json({ message: "No ecosystem account found. Check your Trust Layer ID or email." });
    }
    // Only allow ecosystem login for Trust Layer-linked accounts
    if (!user.trustLayerId) {
      return res.status(401).json({ message: "This account is not linked to the Trust Layer ecosystem. Please sign in with your email and password instead." });
    }
    // Auth: try PIN first (numeric, ≤8 chars), then fall back to password
    let authenticated = false;
    if (user.ecosystemPinHash && trimmedCred.length <= 8 && /^\d+$/.test(trimmedCred)) {
      authenticated = await bcrypt.compare(trimmedCred, user.ecosystemPinHash);
    }
    if (!authenticated) {
      authenticated = await bcrypt.compare(trimmedCred, user.passwordHash);
    }
    if (!authenticated) {
      return res.status(401).json({ message: "Invalid credential. Please check your password or ecosystem PIN." });
    }
    // Create session (use your app's existing session logic)
    const sessionToken = generateToken();
    const expiresAt = new Date(Date.now() + SESSION_DURATION_MS);
    await storage.createSession(user.id, sessionToken, expiresAt);
    res.cookie("session_token", sessionToken, {
      httpOnly: true,
      secure: process.env.NODE_ENV === "production",
      sameSite: "lax",
      maxAge: SESSION_DURATION_MS,
      path: "/",
    });
    console.log(`[Ecosystem Login] ${user.email} authenticated via Trust Layer (${user.trustLayerId})`);
    return res.json({ user: sanitizeUser(user) });
  } catch (error) {
    console.error("Ecosystem login error:", error);
    return res.status(500).json({ message: "Login failed. Please try again." });
  }
});
Important: Make sure sanitizeUser strips ecosystemPinHash and passwordHash from the response. Add ecosystemPinHash to whatever field-stripping function you use.

STEP 4: Frontend — Auth page state and handler
Add a third auth mode ("ecosystem") to your login page state:

const [mode, setMode] = useState<"login" | "register" | "ecosystem">("login");
const [ecosystemLoading, setEcosystemLoading] = useState(false);
Add the submit handler:

const onEcosystemLogin = async (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
  setEcosystemLoading(true);
  const form = new FormData(e.currentTarget);
  const identifier = form.get("identifier") as string;
  const credential = form.get("credential") as string;
  try {
    const res = await fetch("/api/auth/ecosystem-login", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ identifier, credential }),
      credentials: "include",
    });
    const data = await res.json();
    if (res.ok) {
      toast({ title: "Welcome back!", description: "Signed in via Trust Layer ecosystem." });
      // Trigger your auth state refresh / redirect here
    } else {
      toast({ title: "Sign In Failed", description: data.message || "Could not verify your ecosystem credentials.", variant: "destructive" });
    }
  } catch {
    toast({ title: "Error", description: "Network error. Please try again.", variant: "destructive" });
  } finally {
    setEcosystemLoading(false);
  }
};

Key design decisions to preserve
Identifier flexibility — Accept both tl-xxxx-xxxx Trust Layer IDs and plain emails. The endpoint checks the tl- prefix to decide which lookup to use, then falls back to email.
PIN vs password detection — If the credential is numeric and ≤8 characters, try PIN hash first. Always fall back to full password hash. This means users never have to specify which type they're entering.
Gate on trustLayerId — Only users with a Trust Layer ID can use this flow. Regular app-only users get a clear message to use email/password instead.
Strip sensitive fields — Always remove ecosystemPinHash and passwordHash from API responses.
Adapt the app name — Replace "Verdara" with your app name in UI copy. Replace the emerald color with your app's brand accent. The "Supported: Happy Eats, TrustHome..." footer text should list all current ecosystem apps.
